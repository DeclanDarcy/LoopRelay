# Milestone 0.2 Slice 0008: Repository Dashboard Field Catalog

Date: 2026-06-26

## Objective

Advance the first Contract Oracle fixture candidate from endpoint-level inventory to field-level review readiness without capturing a golden fixture yet.

## Scope

Contract identity: Repository dashboard.

Endpoint: `GET /api/repositories`.

Projection: `RepositoryDashboardProjection[]`.

Authority: backend-owned projection shape after backend HTTP JSON serialization.

## Evidence Observed

- `src/CommandCenter.Backend/Program.cs` configures HTTP JSON with `JsonSerializerDefaults.Web` and `JsonStringEnumConverter`.
- `src/CommandCenter.Backend/Endpoints/RepositoriesEndpoints.cs` maps `GET /api/repositories` to `IRepositoryProjectionService.GetDashboardAsync()`.
- `src/CommandCenter.Middle/Projections/RepositoryProjectionService.cs` composes repository dashboard fields from repository, planning, execution, artifact, continuity, reasoning, and decision-session sources.
- `src/CommandCenter.Middle/Projections/RepositoryDashboardProjection.cs` defines the dashboard projection fields.
- `src/CommandCenter.Core/Repositories/Repository.cs`, `src/CommandCenter.Core/Projections/RepositoryContinuitySummary.cs`, `src/CommandCenter.Middle/Projections/RepositoryReasoningSummary.cs`, and `src/CommandCenter.Middle/Projections/RepositoryDecisionSessionSummary.cs` define nested dashboard structures.
- `src/CommandCenter.Execution/Models/ExecutionSessionSummary.cs` and `src/CommandCenter.Execution/Primitives/RepositoryExecutionState.cs` define execution summary fields reused by the dashboard.
- `src/CommandCenter.UI/src/types/repositories.ts` and `src/CommandCenter.UI/src/api/repositories.ts` define manual TypeScript consumer shape and command usage.
- `src/CommandCenter.UI/src/devTauriMock.ts` defines dashboard mock construction through `dashboardEntry`.
- `src/CommandCenter.Shell/src/main.rs` defines a Rust dashboard mirror for `list_repositories`.

## Outputs

- Updated `docs/contract-endpoint-catalog.md` with backend JSON serialization observations.
- Added repository dashboard field ownership catalog covering top-level fields and nested summary fields.
- Updated `docs/contracts.md` to mark repository dashboard as passing the pre-fixture field ownership and serialization-observation gate.
- Updated `docs/architectural-capabilities.md` and `docs/architectural-mechanisms.md` to reflect the new Oracle inventory scope.

## Compatibility Finding

The Rust `RepositoryDashboardProjection` mirror includes `reasoningSummary` but omits `decisionSessionSummary`, while backend and TypeScript dashboard contracts include `decisionSessionSummary`.

This is a parallel-truth drift finding. It is not corrected in this slice because Milestone 0.2 is still defining and observing Oracle truth, not migrating shell transport. Later shell passivity or generated-consumer work must retire, generate, or verify the Rust mirror.

## Verification

Documentation and inventory slice only. No build or test commands were required.

Inventory commands used `rg` and direct file reads across backend endpoint, projection, serialization configuration, shell mirror, TypeScript type/API, mock, and backend test surfaces.
