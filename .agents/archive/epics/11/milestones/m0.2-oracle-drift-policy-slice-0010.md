# Milestone 0.2 Slice 0010: Oracle Drift Policy Classification

Date: 2026-06-26

## Objective

Harden the repository dashboard Contract Oracle pilot so recursive JSON comparison classifies drift by architectural policy rather than reporting all differences as undifferentiated fixture mismatch.

## Capability

Canonical Contract Oracle.

## Affected Files

- `tests/CommandCenter.Backend.Tests/ContractOracleFixtureTests.cs`
- `docs/contracts.md`
- `docs/architectural-mechanisms.md`
- `docs/architectural-capabilities.md`

## Implementation

- Replaced the direct recursive equality assertion with a drift comparison model.
- Added structural drift classification for missing fields, value-kind changes, primitive value changes, and array length changes.
- Added compatibility-review drift classification for additive backend fields that are absent from the fixture.
- Added path-specific reviewed compatibility additions so an explicitly reviewed additive field can be allowed without weakening the fixture globally.
- Added backend tests proving missing-field classification, additive-field review failure, and reviewed additive-field allowance.

## Architectural Impact

The Oracle pilot now distinguishes backend contract breakage from additive compatibility work:

- Structural drift remains a hard failure.
- Additive backend fields require explicit fixture or compatibility review.
- Consumer drift remains outside backend fixture authority and must be handled by later consumer verification.

No production contract shape changed in this slice.

## Compatibility Impact

The existing `repository-dashboard.golden.json` fixture is unchanged. The known Rust shell `RepositoryDashboardProjection` omission of `decisionSessionSummary` remains consumer drift evidence and is not treated as Oracle authority.

## Rollback Path

Revert `ContractOracleFixtureTests.cs` to the previous recursive equality helper and remove the slice documentation updates. The repository dashboard fixture would continue detecting shape mismatch but would lose policy classification and reviewed-addition behavior.
