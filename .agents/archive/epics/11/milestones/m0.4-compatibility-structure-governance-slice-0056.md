# M0.4 Compatibility Structure Governance Slice 0056

## Objective

Add narrow governance for compatibility structures so compatibility fields, routes, commands, and mirrors have explicit lifecycle metadata before later migration and retirement work relies on them.

## Capability

Architectural decision governance.

## Invariant

Architectural decisions govern changes to authority, ownership, contracts, transport, runtime, and mechanisms.

Compatibility fields, routes, commands, and mirrors are transitional. They must have owner, consumer list, replacement path, retirement condition, and supporting evidence before acceptance.

## Changes

- Added `docs/compatibility-structure-governance.md`.
- Added `ArchitecturalDecisionGovernanceTests.CompatibilityStructuresRemainGoverned`.
- The guard validates compatibility-field, compatibility-route, compatibility-command, and compatibility-mirror inventory sections.
- The guard requires each compatibility inventory row to include kind, owner, consumers, replacement path, retirement condition, and reachable evidence.
- The guard aligns compatibility routes with the bounded compatibility route list already protected by `BackendEndpointDispositionTests`.
- The guard aligns compatibility command families and mirror entries with `docs/shell-transport-classification.md`.
- Updated `.agents/decisions/decisions.md` with the Slice 0056 evidence target.
- Updated `.agents/milestones/m0.4-decision-governance.md`, `docs/architectural-capabilities.md`, and `docs/architectural-mechanisms.md`.

## Known Limits

- This is governance inventory, not compatibility correctness certification.
- It does not scan every source field, property, method, or markdown occurrence of compatibility terminology.
- It does not prove that listed compatibility structures derive correctly from authoritative structure.
- It does not decide whether a compatibility structure is good or bad.
- It does not retire shell mirrors, route compatibility, or flattened workflow fields.
- Compatibility derivation, generated artifact replacement, passive transport, and debt retirement remain later milestone work.
