# M0.2 Repository Dashboard Request Boundary Slice 0019

Date: 2026-06-26

## Objective

Add narrow request-boundary verification for the repository dashboard pilot before expanding the Contract Oracle to another contract family.

## Capability

Canonical contract Oracle.

## Scope

This slice verifies only the no-argument request shape for the repository dashboard boundary:

- backend `GET /api/repositories`,
- Rust Tauri `list_repositories`,
- TypeScript API `listRepositories`.

It does not introduce a general request-contract framework.

## Changes

- Added `tests/CommandCenter.Backend.Tests/ContractRequestBoundaryTests.cs`.
- Updated `docs/contracts.md` with the repository dashboard request-boundary pilot.
- Updated `docs/architectural-mechanisms.md` with the request-boundary mechanism and remaining gaps.
- Updated `docs/architectural-capabilities.md` to record Slice 0019 as a local pilot extension, not a milestone certification.

## Evidence

The new verifier proves:

- `GET /api/repositories` has no route parameters and no body metadata.
- Rust `list_repositories()` has no command arguments and forwards `GET /api/repositories` without request-body construction.
- TypeScript `listRepositories()` invokes `list_repositories` without an argument object.

## Verification

Passed:

```powershell
dotnet test tests/CommandCenter.Backend.Tests/CommandCenter.Backend.Tests.csproj --filter FullyQualifiedName~ContractRequestBoundaryTests
```

## Known Limits

- Request-boundary coverage remains limited to repository dashboard.
- Non-empty command argument/body verification remains unimplemented.
- Route/query/body compatibility classification remains unimplemented.
- Milestone 0.2 remains active and uncertified globally.
- The repository dashboard pilot was certified locally in Slice 0018; this slice extends the pilot but does not recertify it.

## Rollback

Remove `ContractRequestBoundaryTests.cs` and revert the documentation updates. The repository dashboard response Oracle, consumer verification, artifact freshness verification, and Slice 0018 local certification would remain intact, but request-boundary drift would no longer be executable.
