# M0.4 Active Governance Artifact Validation Slice 0053

## Objective

Add executable validation for active M0.4 governance artifacts so the current decision checkpoint, governance slice evidence, and mechanism evidence links cannot silently lose their minimum structure.

## Capability

Architectural decision governance.

## Invariant

Architectural decisions govern changes to authority, ownership, contracts, transport, runtime, and mechanisms.

Architectural evidence supports decisions, mechanisms, certification, acceptance, and baselines.

## Changes

- Added `ArchitecturalDecisionGovernanceTests.ActiveGovernanceArtifactsKeepRequiredStructureAndEvidenceLinks`.
- The guard verifies `.agents/decisions/decisions.md` starts as a decision checkpoint and keeps `Authorized Decisions` plus `Next Authorized Sequence` sections.
- The guard verifies current M0.4 governance slice evidence files include required sections for objective, capability, invariant, changes, verification, known limits, and rollback.
- The guard verifies M0.4 governance slice evidence references a `dotnet test` verifier command.
- The guard verifies decision-governance evidence links in `docs/architectural-mechanisms.md` resolve to existing M0.4 slice evidence files.
- Updated `.agents/milestones/m0.4-decision-governance.md`, `docs/architectural-capabilities.md`, and `docs/architectural-mechanisms.md`.

## Known Limits

- This slice validates the active decision checkpoint structure, not the full decision-record template for all rotated decision files.
- This slice validates current M0.4 governance evidence section presence and verifier mention, not the full historical evidence corpus.
- This slice validates mechanism evidence links for decision governance only, not bidirectional reachability among all decisions, evidence, capabilities, and mechanisms.
- M0.4 certification remains pending.

## Rollback

No production code, runtime behavior, generated artifact, fixture, or public contract changed.

Rollback path:

1. Remove the new active governance artifact guard.
2. Revert the M0.4 milestone, capability matrix, and mechanism documentation updates for Slice 0053.
3. Remove this evidence file and return M0.4 to the Slice 0052 governance baseline.
