# M1.2 Repository Dashboard Compatibility Wrapper Bridge Slice 0074

## Objective

Bridge the production `RepositoryDashboardProjection` compatibility wrapper to the generated `RepositoryDashboardConsumerCandidateProjection` while keeping existing product imports stable through `src/CommandCenter.UI/src/types/repositories.ts`.

## Scope

- Added governed `RepositoryExecutionState` metadata for `executionSummary.repositoryState` and `executionHistory[].repositoryState`.
- Regenerated `repository-dashboard.contract-ir.json`, `repository-dashboard.generated.ts`, and `repository-dashboard.generated-artifact-freshness.json`.
- Changed `RepositoryDashboardProjection` to alias `RepositoryDashboardConsumerCandidateProjection`.
- Changed repository decision-session summary wrapper aliases to generated candidate subtypes because dashboard and workspace projections share the same wrapper file.
- Refreshed manual artifact freshness manifests for `repository-dashboard` and `repository-workspace` because the Phase 0 verified `repositories.ts` artifact changed.
- Added consumer-verification coverage for generated `repositoryState` semantic compatibility.

## Authority and State Impact

The bridge does not move semantic authority downstream. Repository execution-state and decision-session domains are carried by governed contract metadata into the generated candidate. The TypeScript wrapper now consumes those generated facts instead of maintaining a parallel dashboard shape.

No runtime state ownership changes were introduced.

## Compatibility Impact

Existing production imports remain stable through `../types` and `src/CommandCenter.UI/src/types/repositories.ts`. This slice does not authorize direct product imports from `src/CommandCenter.UI/src/contracts/generated/repository-dashboard.generated.ts` and does not retire the compatibility wrapper.

Because `repositories.ts` is also the verified manual artifact for the repository workspace contract, both repository dashboard and repository workspace manual freshness manifests were refreshed to the new wrapper-file hash.
