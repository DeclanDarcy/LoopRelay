# M0.4 Regression Weakening Guard Slice 0051

Date: 2026-06-26

## Objective

Add the first M0.4 enforcement check for regression weakening by preventing architecture regression tests from being disabled or focused silently.

## Capability

Architectural decision governance.

## Invariant

Architectural mechanisms cannot disappear, weaken, or lose fixture wiring silently.

Architectural decisions govern changes to authority, ownership, contracts, transport, runtime, and mechanisms.

## Changes

- Added `ArchitecturalDecisionGovernanceTests.ArchitectureRegressionTestsAreNotDisabledOrFocused`.
- The guard scans backend architecture xUnit tests for `Fact` or `Theory` `Skip` usage.
- The guard scans frontend architecture Vitest tests for `.skip` and `.only` usage.
- Updated `docs/architectural-capabilities.md` to record the architecture-regression bypass guard.
- Updated `docs/architectural-mechanisms.md` to record the new guard and its known limits.

## Regression Added

`ArchitectureRegressionTestsAreNotDisabledOrFocused` treats disabled or focused architecture regressions as regression weakening. A future exception must go through governance rather than silently changing test-runner behavior.

## Known Limits

- This is source-pattern enforcement for disabled or focused architecture tests only.
- It does not detect deleted tests, narrowed assertions, removed rows from required catalog arrays, new shell response mirrors, or new compatibility fields.
- It does not yet validate active decision records or evidence packages against their schemas.
- M0.4 certification remains pending.
