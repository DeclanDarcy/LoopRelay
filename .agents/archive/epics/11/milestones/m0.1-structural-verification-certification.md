## Certification Conditions

- .NET build and backend test commands must be run serially until output path isolation is implemented and verified.
- TypeScript structural verification authority is `npm run build`, because it runs `tsc -b` before Vite.
- Local command-line verification is the accepted verifier authority for Phase 0 until CI is added or separately certified.
- Rust compile verification is accepted only as a compiler health signal, not as proof of shell transport behavior.

## Certification Result

Milestone 0.1 is certified for local structural verification.

Certification does not authorize production architecture migration beyond Phase 0 mechanism and documentation work. The next executable protection should be the shell passive response relay regression, because it converts the largest remaining shell verification gap into a testable invariant before passive transport migration.

## Rollback

If a future verifier fails nondeterministically or loses coverage, this certification must be narrowed to the last passing verifier subset and the failing path must be reclassified as quarantined with owner, risk, and retirement criteria.

If CI is added and disagrees with local verification, local-only certification remains valid only for developer work; CI consistency must be resolved before remote enforcement is treated as authoritative.
