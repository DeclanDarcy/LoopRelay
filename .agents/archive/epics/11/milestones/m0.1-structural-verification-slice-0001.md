## Dependency Graph

```text
dotnet build CommandCenter.slnx
  -> consumes src/**/*.csproj and tests/**/*.csproj
  -> produces bin/obj assemblies
  -> protects C# compile and project reference integrity

dotnet test tests/CommandCenter.Backend.Tests/CommandCenter.Backend.Tests.csproj
  -> consumes backend and domain projects plus test project
  -> produces test results and writes shared obj/bin outputs
  -> protects backend behavior, endpoint shape, projections, workflow, decisions, execution, reasoning, continuity, governance

npm run build
  -> consumes TypeScript source, Vite config, package dependencies
  -> produces dist
  -> protects TypeScript structural integrity and production bundle compilation

npm run lint
  -> consumes TypeScript/React source and ESLint config
  -> protects local static style and React hook rules

npm run test
  -> consumes UI source, jsdom setup, dev Tauri mock, characterization tests
  -> produces Vitest result set
  -> protects UI transport, hooks, presentation adapters, navigation, workspace reachability, explainability, workflow, decisions, execution, reasoning, continuity

npm run test:e2e
  -> starts Vite dev server and drives Chromium through Playwright
  -> produces browser-level E2E results
  -> protects workspace certification route, repository selection, tab switching, command palette, viewport navigation

cargo build / cargo test
  -> consumes Rust shell source and Cargo dependencies
  -> produces shell binary/test harness
  -> protects shell compilation; test behavioral coverage is currently absent
```

Important dependency observation: `dotnet build` and `dotnet test` should not be run concurrently against the same configuration because they contend for shared `obj` outputs. A parallel run produced `CS2012` for `CommandCenter.Core.dll`; rerunning `dotnet test` serially passed.
