# Milestone 0.3 Shell Regression Classification Slice 0046

Date: 2026-06-26

## Objective

Add the M0.3 shell regression classification inventory and a minimal guard without changing shell runtime behavior.

## New State

- Added `docs/shell-transport-classification.md`.
- Classified shell command-family responsibilities as `Passive transport`, `Shell-owned operations`, `Transitional compatibility`, or `Unknown / requires review`.
- Recorded Rust mirror properties as orthogonal current and target states:
  - current state: `Passive`, `Mirror`, `Compatibility`, or `Unknown`,
  - target state: `Passive`, `Shell-owned`, `Retired`, or `Quarantined`.
- Added `ShellTransportClassificationDefinesCommandFamiliesAndMirrorInventory` to `tests/CommandCenter.Backend.Tests/Architecture/ArchitecturalRegressionFrameworkTests.cs`.
- Updated `docs/architectural-mechanisms.md` to include the shell classification evidence, category model, and mirror property model.
- Updated `docs/architectural-capabilities.md` to include shell classification as M0.3 protected framework scope.
- Updated `.agents/milestones/m0.3-regression-framework.md` to name shell command-family classification as an accepted mechanism-selection form.

## Classification Findings

- Shell-owned operations are limited to sidecar lifecycle, backend metadata/health, and native repository-directory selection.
- Many workflow, decision, reasoning, continuity, decision-session, and report commands already use `serde_json::Value` and are classified as passive transport for inventory purposes.
- Repository dashboard/workspace, artifact rotation, execution summary, Git status, commit preparation, push result, and request body structs remain transitional compatibility mirrors or adapters.
- `ErrorResponse` remains a compatibility adapter because the shell parses backend errors to preserve boundary-violation envelopes while still returning legacy string errors.

## Known Limits

- This slice does not certify passive transport behavior.
- This slice does not retire Rust mirrors.
- POST body preservation, non-boundary error semantics, status preservation, and domain mirror allowlist enforcement remain later M1.3/passive-transport work.

## Rollback

Remove `docs/shell-transport-classification.md`, the shell classification test additions, and the docs/capability references. Runtime behavior is unchanged.
