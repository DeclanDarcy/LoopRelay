# M0.4 Governance Definition Slice 0050

Date: 2026-06-26

## Objective

Open Milestone 0.4 by defining architectural decision governance, architectural evidence rules, the decision record template, and the first executable metadata guard.

## Capability

Architectural decision governance.

## Invariant

Architectural decisions govern changes to authority, ownership, contracts, transport, runtime, and mechanisms.

Architectural evidence supports decisions, mechanisms, certification, acceptance, and baselines.

- Added `docs/architectural-evidence.md`.

## Decision Classes Defined

- New authority.
- New projection.
- Contract change.
- Compatibility exception.
- Regression weakening.
- Generated artifact exception.
- Transport exception.
- State ownership change.
- Controller/workspace boundary change.
- Runtime failure scope change.
- Reference architecture change.

## Evidence Model Defined

The evidence model now defines:

- evidence package schema,
- evidence type taxonomy,
- evidence requirements by decision class,
- retention rules,
- traceability rules,
- certification standards,
- acceptance standards.

## Regression Added

`ArchitecturalDecisionGovernanceTests` verifies:

- required decision classes exist and include applicability, evidence, regression, and durable documentation obligations,
- mechanism lifecycle approval requirements exist for adding, strengthening, weakening, quarantining, replacing, and retiring mechanisms,
- the evidence package schema and evidence type taxonomy are present,
- the decision record template captures decision metadata, evidence, alternatives, compatibility impact, regression impact, rollback, baseline updates, and follow-up.
