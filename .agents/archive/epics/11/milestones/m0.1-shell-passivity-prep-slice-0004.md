# Milestone 0.1 Follow-on: Shell Passivity Preparation Slice 0004

Date: 2026-06-26

## Objective

Seed the first executable shell transport invariant before beginning Milestone 0.2.

## Capability

- Structural verification follow-on protection.
- Passive transport preparation for Milestones 0.3, 1.2, and 1.3.

## Changes

- Added a testable `backend_get_value_from` helper while keeping production `backend_get_value` callers on `BACKEND_URL`.
- Added Rust unit test `backend_get_value_relays_opaque_json_without_interpretation`.
- Updated `docs/architectural-mechanisms.md` with the passive transport invariant matrix.
- Updated `docs/architectural-capabilities.md` to mark Rust shell behavioral coverage as partial instead of absent.

## Evidence

The shell regression uses a local HTTP fixture and asserts exact `serde_json::Value` equality for a successful backend JSON response containing:

- unknown nested fields,
- nested arrays and objects,
- explicit null values,
- empty strings,
- empty arrays,
- empty objects,
- enum-like backend-owned strings.

## Verification

- `cargo fmt` in `src/CommandCenter.Shell`: passed.
- `cargo test` in `src/CommandCenter.Shell`: passed, 1 test.
- `cargo build` in `src/CommandCenter.Shell`: passed.

## Known Limits

- Protection currently covers only successful JSON relay through the generic GET value helper.
- Backend error-envelope relay is not yet protected.
- Generic POST relay is not yet protected.
- Domain-shaped Rust mirrors still exist and require command-family classification before retirement.

## Rollback

Revert this slice to return Rust shell behavioral coverage to the Milestone 0.1 certified baseline where `cargo test` executes the harness but does not protect shell behavior.
