# Milestone 2 Deep Dive — Canonical Transition Runtime

# Purpose

Milestone 2 creates the canonical runtime that executes every transition in every workflow.

This milestone is **not** about workflows.

It is **not** about Roadmap.

It is **not** about Plan.

It is **not** about Execute.

It is about creating one rigorous, elegant, deterministic transition execution model capable of executing every workflow transition.

When this milestone is complete, workflows become declarative descriptions of domain behavior while the runtime owns orchestration behavior.

---

# Objectives

Implement one canonical runtime responsible for:

* transition lifecycle
* input resolution
* gate evaluation
* prompt execution
* output interpretation
* validation
* effect execution
* persistence
* evidence
* recovery
* successor determination

without embedding workflow-specific logic.

---

# Non-Goals

This milestone does **not**:

* migrate workflows
* implement workflow chaining
* unify CLI
* implement automatic workflow selection
* implement stage resolution
* redesign prompts
* change workflow behavior
* change repository semantics

The runtime exists before workflows migrate onto it.

---

# Architectural Principles

## Runtime owns mechanics

The runtime owns:

* orchestration
* execution
* validation
* persistence
* evidence
* effects
* recovery
* transition lifecycle

Transitions own only domain behavior.

---

## Declarative transitions

A transition should describe:

* what it needs
* what prompt it runs
* what it produces
* how success is determined

It should not orchestrate its own execution.

---

## Gates are deterministic

A transition never advances because a prompt finished.

It advances because:

* output products exist
* output products validate
* output gate succeeds

---

## Every transition is explainable

Every transition execution should be reconstructable from durable evidence.

---

## Repository-first

The runtime reasons only about repository-owned products.

Never hidden process state.

---

# Success Criteria

Milestone 2 is complete when:

* one canonical transition runtime exists
* transitions are declarative
* runtime owns orchestration mechanics
* input resolution is unified
* gate evaluation is unified
* prompt rendering is isolated
* prompt execution is isolated
* output interpretation is unified
* output gates determine completion
* effects execute separately from transitions
* persistence is unified
* recovery is unified
* evidence is unified
* successor resolution is deterministic
* no workflow-specific logic exists inside the runtime
* no workflow has yet migrated

The runtime should now be capable of executing any workflow transition while remaining completely unaware of Roadmap, Plan, Execute, or future workflows.
