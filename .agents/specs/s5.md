# Milestone 5 Deep Dive - HITL Epic Completion Review

## Purpose

Milestone 5 introduces the human review workflow for confirmed non-implementation files at epic completion.

The workflow lets the HITL decide what to keep, delete, or preserve as synthesis after LoopRelay has identified candidate files and optionally extracted useful insight.

---

## Objective

Present the review set to the HITL at epic completion and record explicit decisions.

This milestone answers:

> Which non-implementation files or syntheses should remain because the HITL specifically chose them?

---

## Responsibilities

Milestone 5 owns:

* assembling the epic completion review set from ledger state
* presenting confirmed non-implementation files
* presenting false positives separately for transparency
* presenting uncertain candidates for resolution
* presenting optional free-form synthesis
* recording HITL decisions
* preserving explicit HITL authority over kept non-implementation artifacts

---

## Boundaries

Milestone 5 does not own:

* deterministic classification
* semantic confirmation
* synthesis generation
* autonomous repository acceptance
* commit gating
* publication gating
* planning prompt guidance

HITL review is a review and decision workflow. It is not a general repository mutation acceptance lifecycle.

---

## Implementation Intent

Review should occur at epic completion, matching the intended use of the capability in LoopRelay's epic implementation loop.

The HITL should be able to choose any combination of:

* keep a non-implementation file
* delete a non-implementation file
* keep the synthesis
* discard the synthesis
* resolve an uncertain candidate

The system should preserve the distinction between files generated autonomously and files specifically requested by the HITL. Documentation remains allowed when the HITL requested it.

---

## Review Inputs

The review workflow consumes:

* confirmed non-implementation ledger entries
* false-positive ledger entries
* uncertain candidate ledger entries
* optional free-form synthesis
* semantic rationale and classification evidence where useful

---

## Acceptance Criteria

Milestone 5 is complete when:

* review occurs at epic completion
* confirmed non-implementation files are visible to the HITL
* false positives are visible separately or otherwise auditable
* uncertain candidates can be resolved
* optional synthesis informs keep/delete decisions
* HITL decisions are recorded durably
* kept non-implementation artifacts are tied to explicit HITL authority

---

## Dependencies

Depends on Milestone 3 for:

* ledger state
* false-positive separation
* duplicate suppression

Depends on Milestone 4 for:

* optional free-form synthesis

---

## Failure Modes

Avoid:

* forcing HITL review before epic completion unless explicitly needed
* making review a commit or publication gate
* hiding false positives from review state
* treating autonomous documentation as authorized without HITL intent
* requiring the HITL to rediscover information already summarized by synthesis
