# Milestone 3 Deep Dive — Workflow and Stage Resolution

# Purpose

Milestone 3 transforms the Canonical Transition Runtime into a repository-aware orchestration engine.

Where Milestone 2 answers:

> **How does a transition execute?**

Milestone 3 answers:

> **What should execute next?**

This milestone is responsible for discovering the current repository state and deterministically resolving:

* workflow identity
* workflow eligibility
* current stage
* eligible transitions
* repository authority
* blockers
* ambiguity

It performs **no workflow chaining**.

It does **not** migrate workflows.

It determines the current execution position only.

---

# Objectives

Create one canonical resolution engine responsible for:

* automatic storage verification
* repository observation
* workflow identity resolution
* workflow eligibility resolution
* stage resolution
* transition eligibility
* ambiguity detection
* blocker detection
* explainability

The runtime must produce deterministic orchestration decisions from repository state.

---

# Non-Goals

This milestone does **not**:

* execute workflows
* chain workflows
* migrate Roadmap
* migrate Plan
* migrate Execute
* redesign persistence
* redesign prompts
* redesign recovery

Those are handled by other milestones.

---

# Architectural Principles

## Repository is the source of truth

Resolution uses repository-owned information only.

No hidden runtime state.

No cached orchestration decisions.

Every decision must be reproducible.

---

## Resolution is deterministic

Given identical repository state,

resolution must produce identical results.

---

## Ambiguity is explicit

If repository evidence cannot uniquely determine a decision,

the runtime reports ambiguity.

It must never silently guess.

---

## Resolution is non-mutating

Stage resolution must never mutate the repository.

Storage verification is automatic.

Storage repair is not.

---

## Explainability is mandatory

Every orchestration decision must identify:

* authoritative evidence
* supporting evidence
* ignored evidence
* conflicting evidence
* uncertainty

# Success Criteria

Milestone 3 is complete when:

* repository observation is unified
* storage verification is automatic
* workflow identity resolves deterministically
* explicit workflow identities are supported
* default Eval versus Traditional selection works
* workflow eligibility is explicit
* workflow progress is reconstructable
* stage resolution is deterministic
* transition eligibility is deterministic
* blockers are explicit
* ambiguity is explicit
* every orchestration decision is explainable
* repository state has one canonical classification
* no repository mutation occurs during resolution
* no workflow execution occurs during resolution

At completion, the runtime knows **what** can execute and **why**, but has not yet begun executing or chaining workflows. That foundation enables Milestone 4 to introduce workflow chaining and unified workflow boundaries without embedding discovery logic into the workflows themselves.
