# Decisions

## Newly Authorized

- Treat Milestone 1 implementation work as effectively complete.
- Treat `Workstream 1.5 complete` as the important milestone outcome, not maximizing `Button` or primitive adoption counts.
- Move next into a formal Milestone 1 certification review before opening Milestone 2.
- Certify Milestone 1 against visual consistency, theme consistency, primitive authority, behavioral equivalence, and remaining duplication.
- Require primitive authority to remain render-only, with no workflow interpretation, readiness decisions, or navigation ownership.
- Explicitly state during certification whether workflow behavior remained unchanged.
- Treat remaining primitive opportunities as in-scope only when they are meaningful architectural improvements, not mechanical primitive-count optimization.
- Use the M2 readiness question as the transition gate: if the remaining problem is visual consistency, stay in M1; if it is application shell architecture, workspace composition, layout strategy, or information architecture, transition to M2.
