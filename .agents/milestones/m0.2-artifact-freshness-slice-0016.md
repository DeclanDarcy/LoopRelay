## Architectural Impact

This slice introduces a Phase 0 artifact freshness mechanism without creating a production generator or moving Milestone 1.2 responsibilities into Milestone 0.2.

The mechanism is intentionally separate from:

- Oracle fixture comparison, which checks backend serialization against accepted backend-owned fixture truth.
- Consumer verification, which checks downstream shape conformance to Oracle-observed truth.

Artifact freshness checks whether a tracked contract artifact baseline has moved in lockstep with the Oracle source that justifies it.

## Protected Failure Modes

- Stale generated artifact: Oracle source changes while the artifact remains on the prior baseline, or both source and artifact change without refreshed regeneration evidence.
- Unexpected manual artifact modification: artifact changes while the Oracle source baseline does not.
- Missing expected artifact: manifest names an artifact that no longer exists.

## Known Limits

- Coverage remains limited to the repository dashboard pilot fixture.
- The verified artifact is manual TypeScript, not generated output.
- The mechanism hashes whole files and does not yet prove deterministic generation.
- The mechanism does not write artifacts, verify generated headers, compare command argument artifacts, or classify semantic reinterpretation.
- Oracle certification remains pending.
