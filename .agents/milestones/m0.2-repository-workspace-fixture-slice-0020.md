# Milestone 0.2 Slice 0020: Repository Workspace Fixture Pilot

Date: 2026-06-26

## Objective

Prove the Contract Oracle pilot can repeat across a second contract family by adding repository workspace field inventory and a golden backend serialization fixture.

## Scope

Contract identity: Repository workspace.

Primary endpoint: `GET /api/repositories/{repositoryId}/workspace`.

Related producers: `POST /api/repositories/{repositoryId}/refresh`, current handoff rotation, and current decisions rotation, because they also return `RepositoryWorkspaceProjection`.

Projection: `RepositoryWorkspaceProjection`.

Authority: backend-owned projection shape after backend HTTP JSON serialization.

## Evidence Observed

- `src/CommandCenter.Backend/Endpoints/RepositoriesEndpoints.cs` maps repository workspace and refresh routes to `IRepositoryProjectionService`.
- Artifact rotation endpoints return refreshed `RepositoryWorkspaceProjection` values after mutation.
- `src/CommandCenter.Middle/Projections/RepositoryWorkspaceProjection.cs` defines the workspace projection fields.
- `src/CommandCenter.Middle/Projections/RepositoryProjectionService.cs` composes workspace fields from repository, planning, execution, artifact, continuity, reasoning, and decision-session sources.
- `src/CommandCenter.Core/Projections/ArtifactInventory.cs` and `src/CommandCenter.Core/Artifacts/Artifact.cs` define workspace artifact inventory shape.
- `src/CommandCenter.Continuity/Models/OperationalContextProjection.cs`, `OperationalContextProposalSummary.cs`, and `OperationalContextItem.cs` define the full operational-context workspace shape.
- `src/CommandCenter.UI/src/types/repositories.ts`, `artifacts.ts`, and `operationalContext.ts` define manual TypeScript consumer shape.
- `src/CommandCenter.UI/src/devTauriMock.ts` returns workspace mock entries from `state.workspaces`.
- `src/CommandCenter.Shell/src/main.rs` defines a Rust workspace mirror for repository workspace commands.

## Fixture Coverage

The workspace fixture intentionally covers:

- stable repository identity, name, and path,
- string enum serialization,
- explicit nulls,
- populated execution summary and history,
- accepted execution, commit, and push metadata,
- populated artifact inventory arrays,
- explicit `currentDecisions: null`,
- populated operational-context item arrays,
- operational-context proposal status, timestamps, counts, and null promotion fields,
- populated reasoning summary,
- populated decision-session summary,
- empty nested arrays for decision-session findings, recent transfer lineage, and diagnostics,
- and the `decisionSessionSummary` field that is missing from the Rust workspace mirror.

## Compatibility Finding

The Rust `RepositoryWorkspaceProjection` mirror includes `reasoningSummary` but omits `decisionSessionSummary`, while backend and TypeScript workspace contracts include `decisionSessionSummary`.

This is a downstream consumer drift finding. It is intentionally not corrected in this slice because Milestone 0.2 is proving Oracle truth and fixture repeatability, not retiring shell response mirrors.
