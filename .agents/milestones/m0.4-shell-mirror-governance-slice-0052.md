# M0.4 Shell Mirror Governance Slice 0052

## Objective

Add executable governance for new or stale Rust shell mirror structs so shell transport inventory cannot drift from `src/CommandCenter.Shell/src/main.rs`.

## Capability

Architectural decision governance.

## Invariant

Transport preserves request, response, status, null, empty, and error semantics without domain participation. New shell response mirrors are transport responsibility growth and require governance.

## Changes

- Added `ArchitecturalDecisionGovernanceTests.ShellRustStructsRemainClassifiedInTransportInventory`.
- The guard scans Rust `struct` declarations in `src/CommandCenter.Shell/src/main.rs`.
- The guard parses the Rust Mirror Inventory in `docs/shell-transport-classification.md`.
- The guard fails when a Rust struct exists without inventory classification.
- The guard fails when an inventory struct no longer exists in Rust source.
- Updated `.agents/milestones/m0.4-decision-governance.md`, `docs/architectural-capabilities.md`, and `docs/architectural-mechanisms.md`.

## Known Limits

- This slice verifies inventory presence and stale entry detection only.
- It does not certify that every classified struct has the correct current state or target state.
- It does not retire any existing shell mirror.
- It does not certify passive transport behavior.

## Rollback

Remove the new guard and revert documentation updates. If the guard blocks legitimate shell-owned additions, classify the struct in `docs/shell-transport-classification.md` with owner, target state, and retirement or keep condition instead of bypassing the guard.
