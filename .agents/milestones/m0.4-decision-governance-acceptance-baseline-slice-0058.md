- architecture-affecting change classes are defined with decision rules, evidence obligations, compatibility impact, regression impact, rollback, and durable documentation requirements,
- architectural evidence packages are required for decisions, mechanisms, certification, acceptance, rollback, and baseline updates,
- compatibility exceptions require owner, consumers, replacement path, retirement condition, and reachable evidence before broad migration begins,
- rollback policy and rollback evidence requirements are durable before generated contracts, passive transport, authority restoration, and frontend ownership migrations begin,

- regression weakening, shell mirror inventory drift, authority/projection-like file names, and compatibility structures have initial governance detectors.

Accepted deferrals:

- complete historical decision and evidence schema validation,
- automated semantic quality review for human-authorized decisions,
- complete source-change detection for every possible architecture-affecting edit,
- compatibility derivation correctness and retirement readiness,
- passive transport certification and shell mirror retirement,
- generated contract ecosystem certification,
- semantic authority restoration, state ownership, controller/workspace architecture, runtime isolation, CI enforcement, and release-path certification.
