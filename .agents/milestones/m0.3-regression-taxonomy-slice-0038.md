# Milestone 0.3 Regression Taxonomy Slice 0038

Date: 2026-06-26

## Objective

Define the M0.3 regression taxonomy so future architectural invariants choose protections from a shared mechanism vocabulary instead of adding ad hoc checks.

## Capability

Architectural regression framework.

## Changes

- Replaced the informal initial taxonomy in `docs/architectural-mechanisms.md` with `### Regression Taxonomy`.
- Added taxonomy metadata for category, preferred mechanism, minimum acceptable mechanism, preferred execution phase, owner, severity, evidence, drift model, and remediation.
- Added severity rules for advisory warning, compatibility warning, local build failure, CI failure, and release blocker.
- Added `RegressionTaxonomyDefinesMechanismSelectionMetadata` to `tests/CommandCenter.Backend.Tests/Architecture/ArchitecturalRegressionFrameworkTests.cs`.
- Updated `.agents/milestones/m0.3-regression-framework.md` to mark the regression taxonomy output complete.
- Updated `docs/architectural-capabilities.md` to record the taxonomy guard as an active M0.3 protection.

## Authority And State Impact

No production authority or mutable state moved. This slice strengthens the architecture-test metadata used to choose future protection mechanisms.

## Compatibility Impact

No runtime or contract compatibility impact. The new taxonomy records compatibility warning as a severity class for future consumer and compatibility-field drift.

## Rollback Path

Revert this slice's documentation and test changes. The previous invariant catalog and M0.3 skeleton remain valid, but later slices would lose the protected mechanism-selection vocabulary until a replacement taxonomy is added.
