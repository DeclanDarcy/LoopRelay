# Milestone 0.2: Establish the Canonical Contract Oracle

Acceptance status: accepted as a scoped Phase 0 Contract Oracle foundation in `.agents/milestones/m0.2-oracle-acceptance-baseline-slice-0035.md`.

This acceptance does not certify full contract-surface coverage. It freezes the M0.2 baseline around the Oracle architecture, governance, lifecycle, repeatability, and three locally certified pilots: repository dashboard, repository workspace, and primary workflow projection. Generated contracts, mechanical versioning, automatic regeneration, complete dependency graph coverage, passive transport certification, and full contract catalog coverage are deferred to later milestones.

Objective: create a single observation point for cross-boundary contract truth.

Implementation tasks:

- [x] Inventory every cross-boundary contract at family level and endpoint-catalog level; field-level inventory is complete for certified pilots only.
- [x] Define the canonical contract authority for Command Center as the serialized external shape of backend-owned projections and command results under the backend JSON configuration.
- [x] Define boundary taxonomy: domain, projection, contract, serialization, transport, resource, controller, presentation, persistence, runtime.
- [x] Define serialization rules for identifiers, enum strings, dates, nulls, optional fields, collections, ordering, naming, error envelopes, streams, versioning, and compatibility fields at the Phase 0 Oracle level.
- [x] Add golden serialized fixtures for the accepted representative pilots: repository dashboard, repository workspace, and primary workflow projection.
- [x] Add recursive contract comparison tests that compare backend serialization against accepted pilot fixtures and fail on unreviewed drift.
- [x] Add an Oracle change workflow: contract identity, compatibility classification, fixture update, consumer/artifact action, verification, rollback, and review evidence.
- [x] Identify parallel truth classes across C#, Rust, TypeScript, mocks, fixtures, docs, and tests, with pilot-specific drift evidence.

- [x] Canonical contract inventory, accepted at family/catalog level with pilot field detail.
- [x] Oracle specification.
- [x] Boundary taxonomy.
- [x] Serialization architecture.
