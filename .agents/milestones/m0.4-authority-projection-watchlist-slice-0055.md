# M0.4 Authority and Projection Watchlist Slice 0055

## Objective

Add narrow ungoverned-change detection for new authority-like and projection-like source file names so new named architectural artifacts cannot appear silently before the semantic authority and projection taxonomy milestones.

## Capability

Architectural decision governance.

## Invariant

Architectural decisions govern changes to authority, ownership, contracts, transport, runtime, and mechanisms.

Backend domain services compute semantic meaning, and projections expose authoritative meaning without becoming new semantic authority.

## Changes

- Added `docs/authority-projection-governance-watchlist.md`.
- Added `ArchitecturalDecisionGovernanceTests.AuthorityAndProjectionLikeFileNamesRemainGoverned`.
- The guard scans `src/` and `tests/CommandCenter.Backend.Tests/` for `.cs`, `.ts`, `.tsx`, and `.rs` file names containing `Authority` or `Projection`.
- The guard requires every watched file to appear in the authority/projection watchlist and rejects stale watchlist entries.
- Updated `.agents/milestones/m0.4-decision-governance.md`, `docs/architectural-capabilities.md`, and `docs/architectural-mechanisms.md`.

## Known Limits

- This is a file-name detector only.
- It does not scan type names, method names, field names, markdown, generated output, or archived evidence.
- It does not prove that listed authorities are correct or that listed projections are pure.
- It does not detect downstream semantic inference when the file name lacks `Authority` or `Projection`.
- Existing entries are inventory-level governance, not semantic certification.
