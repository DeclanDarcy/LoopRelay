# Milestone 0.2 Slice 0009: Repository Dashboard Golden Fixture Pilot

Date: 2026-06-26

## Objective

Create the first executable Contract Oracle protection by capturing a repository dashboard golden fixture and comparing backend JSON serialization against it recursively.

## Scope

Contract identity: Repository dashboard.

Endpoint: `GET /api/repositories`.

Projection: `RepositoryDashboardProjection[]`.

Authority: backend-owned projection shape after backend JSON serialization.

## Outputs

- Added `tests/CommandCenter.Backend.Tests/ContractFixtures/repository-dashboard.golden.json`.
- Added `ContractOracleFixtureTests.RepositoryDashboardGoldenFixtureMatchesBackendSerialization`.
- Updated the backend test project to copy contract fixture JSON into the test output.
- Updated `docs/contracts.md`, `docs/architectural-mechanisms.md`, and `docs/architectural-capabilities.md` to mark the Oracle as partially executable but uncertified.

## Fixture Coverage

The first dashboard fixture intentionally covers:

- stable repository identity, name, and path,
- string enum serialization,
- explicit nulls,
- populated execution summary,
- populated execution history,
- timestamp serialization,
- `TimeSpan` duration serialization,
- populated continuity summary,
- populated reasoning summary,
- populated decision-session summary,
- nested arrays and nested objects,
- and the `decisionSessionSummary` field that is missing from the Rust dashboard mirror.

Known limitation: this first fixture does not include an empty dashboard collection or an empty nested collection on the dashboard object. Empty collection serialization still needs a second dashboard variant or another representative fixture before Milestone 0.2 certification.

## Recursive Comparison Behavior

The pilot comparison:

- treats object property order as non-semantic,
- treats array order and length as contract-significant,
- fails on missing fields,
- fails on unexpected fields,
- fails on JSON kind changes, including null/object and array/scalar changes,
- fails on value changes,
- and reports path-scoped structural drift messages for object and array shape failures.

Policy-drift classification is not implemented yet. Fixture updates currently require explicit review through milestone evidence.

## Compatibility Finding Preserved

The Rust `RepositoryDashboardProjection` mirror still omits `decisionSessionSummary`. The new fixture makes this backend-owned field executable Oracle evidence. It does not authorize Rust, TypeScript, mocks, or characterization tests as contract authority.
