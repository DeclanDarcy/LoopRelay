# Milestone 1.2: Build the Generated Contract Ecosystem

Objective: replace parallel hand-maintained contract truths with deterministic generated or mechanically verified artifacts.

Current slice state:

- Slice 0068 starts M1.2 as generation-pipeline validation, not consumer migration.
- The first pilot family is `repository-dashboard`, reusing the accepted Phase 0 Oracle fixture.
- The executable path is Oracle fixture -> canonical generation IR -> generated TypeScript contract metadata and raw aliases -> freshness manifest.
- Slice 0069 extends the generated artifact with raw TypeScript aliases and verifies `RepositoryDashboardGeneratedProjection` against the Oracle fixture.
- No production manual TypeScript consumer, Rust shell mirror, dev Tauri mock, API wrapper, or UI import has been migrated by these slices.
- The generated raw aliases are fixture-observed rather than schema-complete; production consumer migration remains later M1.2 work after nullability and semantic alias policy are governed.
