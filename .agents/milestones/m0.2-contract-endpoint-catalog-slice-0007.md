## Architectural Result

This slice makes endpoint routes, consumers, compatibility consumers, and first-fixture priorities visible without changing production behavior.

The catalog records a required endpoint-level inventory shape:

- endpoint or command,
- backend authority,
- projection type,
- serialization authority,
- contract identity,
- consumers,
- compatibility consumers,
- fixture candidate,
- fixture priority.

It also defines narrow observable serialization rules for fixture preconditions:

- backend-emitted property names are the contract names,
- GUIDs and string identifiers remain strings,
- enum-like lifecycle states are backend-owned strings,
- explicit null, omitted properties, empty collections, empty objects, and empty strings are distinct observable states,
- date/time format must be captured from backend serialization,
- ordering must be classified as semantic or incidental,
- unknown fields and backend error envelopes must be preserved through transport,
- streams need stream-specific fixture or trace rules,
- compatibility fields require owner, consumers, replacement path, and retirement condition.
