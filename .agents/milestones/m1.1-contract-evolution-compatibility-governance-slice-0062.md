## Architecture Impact

This slice adds durable M1.1 definitions for:

- contract evolution operations, including additive field, deprecated field, removed field, renamed field, semantic reinterpretation, representation normalization, contract split, contract merge, projection split, projection aggregation, and compatibility-only alias;
- identity, compatibility, version, consumer-action, and evidence expectations for each evolution operation;
- compatibility states for compatible baseline, reviewed additive, bridged compatibility, consumer-stale, incompatible accepted, and quarantined exception;
- compatibility obligations for fields, aliases, routes, commands, mirrors, manual artifacts, dev mocks, and shell Rust mirrors;
- a contract governance sequence from identity classification through compatibility review, artifact action, verification, evidence, and handoff update.

The slice keeps M1.1 model-first. It does not add generation, change fixtures, alter endpoint behavior, migrate Rust mirrors, migrate TypeScript types, or expand dev Tauri mock coverage.

## Compatibility Impact

No runtime compatibility surface changed. Existing compatibility obligations remain:

- Rust repository dashboard and workspace mirrors still have known `decisionSessionSummary` drift.
- Manual TypeScript repository and workflow types remain verified compatibility artifacts, not contract authority.
- Dev Tauri mock repository dashboard and workspace payloads remain verified compatibility consumers.
- Dev mock workflow response coverage remains absent.

The new compatibility model defines how those obligations must be reviewed, bridged, quarantined, migrated, and retired before M1.2 generation and M1.3 transport work.

## Governance

This slice follows the active decision to continue M1.1 with evolution, compatibility, and governance before canonical examples. It preserves the separation of stability, compatibility, and versioning: stability classifies identity significance, compatibility records consumer obligations, and versioning records accepted lifecycle state.
