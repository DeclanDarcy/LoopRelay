## Architecture Impact

This slice defines how contract identities are classified and owned before the generated ecosystem exists. It makes the following M1.1 model elements durable:

- contract taxonomy for public projection, internal projection, command request, command response, event, notification, streaming event, stream lifecycle, persistence, configuration, diagnostics, health, certification, error envelope, and compatibility artifact categories;
- projection-to-contract relationship rules for single projection contracts, multi-identity projections, shared identities, aggregation contracts, request/response separation, stream lifecycles, and compatibility fields;
- allowed and forbidden transformations at the projection-to-contract boundary;
- ownership dimensions for semantics, shape, serialization, compatibility, versioning, evolution, deprecation, and consumers;
- consumer classes for generated, verified, observational, transforming, testing, presentation, and compatibility consumers.

No runtime code changed. No endpoint behavior, fixture, generated artifact, shell command, TypeScript type, or dev mock changed.
