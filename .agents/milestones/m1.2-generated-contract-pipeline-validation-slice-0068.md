# M1.2 Generated Contract Pipeline Validation Slice 0068

## Objective

Start Milestone 1.2 by proving a narrow generated-contract pipeline without migrating consumers or redefining the accepted M1.1 contract model.

## Scope

- Pilot contract family: `repository-dashboard`.
- Source authority: `tests/CommandCenter.Backend.Tests/ContractFixtures/repository-dashboard.golden.json`.
- Generated IR: `tests/CommandCenter.Backend.Tests/ContractFixtures/repository-dashboard.contract-ir.json`.
- Generated TypeScript metadata: `src/CommandCenter.UI/src/contracts/generated/repository-dashboard.generated.ts`.
- Freshness manifest: `tests/CommandCenter.Backend.Tests/ContractFixtures/repository-dashboard.generated-artifact-freshness.json`.

## Implementation

- Added `ContractGeneratedArtifactPipelineTests`.
- Added `ContractGenerationSupport` for Oracle fixture -> canonical IR -> TypeScript metadata generation.
- Added explicit update mode gated by `COMMANDCENTER_UPDATE_GENERATED_CONTRACTS=1`.
- Added generated artifact freshness coverage for the generated repository dashboard TypeScript metadata artifact.
- Updated `docs/contracts.md`, `docs/architectural-capabilities.md`, and `.agents/milestones/m1.2-generated-contracts.md`.

## Architectural Claim

The first M1.2 slice proves that generation can consume an accepted Oracle fixture, produce deterministic IR and TypeScript metadata, and fail verification when the generated artifact changes without a source change. It does not certify full TypeScript consumer generation, mock generation, Rust metadata generation, command-body schemas, or versioning.

## Compatibility

No product consumer was migrated. Existing manual TypeScript types, Rust mirrors, and dev Tauri mock payloads remain compatibility or verified consumers under the existing Phase 0/M1.1 protections.

## Verification

Passed:

```powershell
$env:COMMANDCENTER_UPDATE_GENERATED_CONTRACTS='1'; dotnet test tests/CommandCenter.Backend.Tests/CommandCenter.Backend.Tests.csproj --filter FullyQualifiedName~ContractGeneratedArtifactPipelineTests; Remove-Item Env:COMMANDCENTER_UPDATE_GENERATED_CONTRACTS
dotnet test tests/CommandCenter.Backend.Tests/CommandCenter.Backend.Tests.csproj --filter "FullyQualifiedName~ContractGeneratedArtifactPipelineTests|FullyQualifiedName~ContractGeneratedArtifactFreshnessTests"
```

Normal check-mode result: 10 passed, 0 failed, 0 skipped.

## Rollback

Delete the generated IR, generated TypeScript metadata, generated freshness manifest, and `ContractGeneratedArtifactPipelineTests`/`ContractGenerationSupport` additions. Existing Phase 0 manual artifact freshness and consumer verification remain intact because no consumer imports were changed.

## Known Limits

- The generated TypeScript artifact is metadata-only.
- Null-only fixture observations generate `null` metadata for the observed path rather than full semantic optionality.
- The update workflow is test-gated, not a standalone generator CLI.
- Only `repository-dashboard` is covered.
