# M0.2 Oracle Repeatability Evidence Slice 0025

Date: 2026-06-26

## Objective

Record cross-pilot evidence that the Milestone 0.2 Contract Oracle lifecycle repeated across repository dashboard and repository workspace without requiring a framework redesign before expanding to a more complex contract family.

## Capability

Canonical contract Oracle.

## Repeatability Claim

The Contract Oracle pilot lifecycle is repeatable at repository read-model scope.

This claim is limited to the repository dashboard and repository workspace pilots. It does not certify Milestone 0.2 globally and does not prove that workflow, decisions, reasoning, execution, Git, streaming, error-envelope, or non-empty command-body contracts are covered.

## Mechanisms Reused Without Redesign

- Contract identity and field-ownership inventory.
- Backend JSON serialization fixture comparison.
- Structural drift versus compatibility-review drift classification.
- Shared recursive consumer verifier and Rust, TypeScript, and dev mock shape providers.
- Phase 0 contract artifact freshness manifest and verifier.
- Request-boundary checks across backend route, Rust Tauri command, and TypeScript API wrapper.
- Local certification evidence format.
- Procedural Oracle change workflow.

## Framework Changes Required

No Oracle framework redesign was required to move from repository dashboard to repository workspace.

The workspace pilot did require additional fixture data, field catalog entries, consumer test cases, freshness manifest entries, and request-boundary checks. Those were contract-family additions, not changes to the Oracle architecture.

## Known Limits

- Repeatability is proven only across two repository read-model contract families.
- Repository workspace request-boundary certification covers only the primary GET path; refresh and artifact rotation request boundaries remain pending.
- Known Rust shell mirror drift remains for dashboard and workspace `decisionSessionSummary`.
- TypeScript artifact freshness still protects a Phase 0 verified manual contract artifact, not generated Milestone 1.2 output.
- Semantic reinterpretation checks, deterministic generation, fixture update automation, mechanical versioning, stream fixtures, error-envelope fixtures, non-empty command-body verification, and broad dependency graph coverage remain pending.
